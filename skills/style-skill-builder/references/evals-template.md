# Evals Template

Use pass/fail checks. Replace bracketed text with style-specific criteria.

## Opening

- [ ] Pass if [opening behavior required by the style]. Fail if [specific failure mode].

## Voice

- [ ] Pass if the draft [observable voice behavior]. Fail if it [specific unwanted voice behavior].
- [ ] Pass if the draft avoids [blacklist pattern]. Fail if [pattern appears without doing real work].

## Structure

- [ ] Pass if the piece moves from [expected starting point] to [expected development] to [expected ending]. Fail if it [structure failure].

## Sentence Level

- [ ] Pass if sentences [rhythm, diction, abstraction criteria]. Fail if they [failure mode].

## Substance

- [ ] Pass if claims are specific and source-backed. Fail if the draft invents facts, cites vague authority, or overstates certainty.

## Channel

- [ ] Pass if the output fits [channel]. Fail if it uses a format, length, or call to action that would feel wrong there.

## Constraints

- [ ] Pass if legal, accessibility, localization, product naming, and external publishing constraints are respected. Fail if any constraint is violated or uncertain without being surfaced.

## Result Format

```markdown
## Eval Results

| Check | Result | Evidence | Fix |
| --- | --- | --- | --- |
| Opening | Pass/Fail | ... | ... |

## Overall

Pass/Fail. If fail, revise and rerun.
```
