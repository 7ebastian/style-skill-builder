# Analysis Framework

Use this when the user asks for analysis, diagnosis, comparison, or a rationale before generating a style guide. Do not force every tier if the user only needs a quick guide.

## Tiers

### Overall Design

- Opening and closing: hook, friction, payoff, lingering tension.
- Global shape: narrative arc, chronology, deductive argument, mosaic, field report, guide, interview, or case study.
- Forward motion: how each section pulls the reader onward.

### Main Idea

- Locate the core claim, question, thesis, or emotional center.
- Explain how sections support, complicate, or drift from it.
- Note whether the piece has one big idea or too many competing ideas.

### Structure

- Identify the movement: example to idea, scene to argument, problem to framework, confession to practice, tension to resolution.
- Name recurring structural templates when possible.
- Flag throat-clearing, buried stakes, weak transitions, and summary endings.

### Sentences and Rhythm

- Note sentence length, variation, pacing, emphasis, openings, closings, fragments, repetition, and punctuation habits.
- Identify what makes a sentence sound like the target voice.
- Flag over-polished, over-symmetrical, abstract, or generic sentences.

### Voice and Tone

- Describe tensions and boundaries, not just adjectives.
- Identify reader relationship: peer, mentor, field reporter, practitioner, narrator, critic, confidant.
- Note emotional temperature and acceptable tonal range.

### Argument and Evidence

- Identify evidence habits: anecdotes, screenshots, metrics, named examples, research, analogy, expert quotes, lived experience.
- Flag unsupported claims, fake authority, vague references, and overconfident generalization.

### Vocabulary and Accessibility

- Identify concrete nouns, verbs, jargon tolerance, abstraction level, and explanation style.
- Note whether technical terms need plain-language treatment.
- Flag generic framing language and inflated vocabulary.

### Signature Moves

- Name repeatable craft moves.
- For each move, state what it does, when to use it, and what failure looks like.

### Quality-Control Markers

- Identify tests that predict whether a draft is working.
- Convert recurring failures into checklist items or evals.

## Output

For analysis only:

1. Executive summary.
2. Point-by-point diagnosis.
3. Reusable patterns.
4. Actionable recommendations.
5. Source confidence and open questions.

For analysis plus guide:

1. Executive summary.
2. Style diagnosis.
3. Reusable patterns.
4. Neutral style guide.
5. Testing recommendations.
